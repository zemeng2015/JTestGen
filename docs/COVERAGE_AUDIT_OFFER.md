# Free Java Coverage Audit

JTestGen is looking for early Java/Maven projects with JaCoCo coverage gaps.

## What the audit provides

- identify low-coverage classes
- identify AI-fixable test targets
- estimate coverage improvement opportunities
- optionally generate one passing test candidate
- produce a before/after report if generation succeeds

## What users need

Option A:

- run JTestGen locally
- share only report.json and high-level results

Option B:

- share a public repo
- share JaCoCo XML path
- share Maven command

## What JTestGen will not do

- modify production source code
- commit directly to your main branch
- guarantee business correctness
- replace human review

## Paid follow-up package

Coverage Improvement PR Package:

- generate tests for 1-3 target classes in the first paid/trial package
- run Maven validation
- repair failures
- provide before/after coverage report
- deliver as PR or patch

Suggested first paid range: `$100-$300`.
